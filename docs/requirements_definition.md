## 1. プロジェクト概要

### 1.1 アプリケーション名

**Tana**（棚）— 治療院・サロン向け在庫管理アプリ

### 1.2 目的

治療院・サロンが日常業務で使用する施術用消耗品と物販商品の在庫を、スマートフォンやタブレットから簡単に管理できるWebアプリケーションを提供する。外部サーバーへの通信を一切行わず、ブラウザ内にすべてのデータを保持することで、インターネット環境に依存しない運用を実現する。

### 1.3 アプリケーション種別

- **PWA（Progressive Web App）** — ホーム画面追加・オフライン動作に対応
- **SPA（Single Page Application）** — 画面遷移なし、タブ切り替えによるUI
- **フロントエンドのみ** — サーバーサイド処理なし、ブラウザ内完結

### 1.4 技術スタック

| 項目 | 技術 |
|------|------|
| フロントエンド | Vanilla JavaScript（ES6+）、HTML5、CSS3 |
| データ保存 | IndexedDB（ブラウザ内永続ストレージ） |
| 計算ロジック | `tana.calc.js`（純粋関数のみ、DOM操作・DB操作禁止） |
| UI操作・DB操作 | `script.js`（すべてのUI操作・IndexedDB操作を集約） |
| バーコード読取 | html5-qrcode ライブラリ（ローカル同梱） |
| PWA基盤 | Service Worker（`sw.js`）、Webマニフェスト（`manifest.json`） |
| 配信 | Docker + nginx:alpine（ポート 8088） |
| テスト | Jest（ユニットテスト）、Docker内E2Eテスト |

### 1.5 ファイル構成

```
local_app/
  index.html          … SPAエントリポイント
  style.css           … スタイルシート
  script.js           … UI操作・DB操作
  tana.calc.js        … 純粋計算関数
  sw.js               … Service Worker
  version.js          … バージョン情報
  manifest.json       … PWAマニフェスト
  lib/
    html5-qrcode.min.js … バーコードスキャンライブラリ
  icons/
    icon-192.png, icon-512.png, icon-maskable-512.png, apple-touch-icon.png, favicon-32.png, favicon-16.png
```

## 2. 対象ユーザーと利用シナリオ

### 2.1 対象ユーザー

以下の業種のオーナーおよびスタッフを主な対象ユーザーとする。

- 治療院（整骨院・接骨院）
- 鍼灸院
- 美容室
- エステサロン
- ネイルサロン
- まつげサロン

### 2.2 想定される利用シナリオ

| シナリオ | 説明 |
|----------|------|
| 日常の入出庫記録 | 商品入荷時に入庫を記録し、施術で使用した際に使用を記録する |
| 物販商品の販売記録 | 顧客に販売した商品の数量と日付を記録する |
| バーコードスキャン入出庫 | 商品のバーコードをスマートフォンのカメラでスキャンし、素早く入庫・使用を記録する |
| 在庫不足の確認 | ダッシュボードで低在庫アラートを確認し、発注すべき商品を把握する |
| 使用期限管理 | 期限が近い商品・期限切れ商品をアラートで把握し、適切に対応する |
| 定期棚卸 | 月次等のタイミングで全商品の実数をカウントし、システム在庫との差異を確認・調整する |
| データバックアップ | JSON形式でデータをエクスポートし、端末故障やブラウザデータ消去に備える |

## 3. システム構成

### 3.1 アーキテクチャ概要

```
[ブラウザ]
  ├── index.html（SPA）
  │     ├── script.js（UI操作・DB操作）
  │     ├── tana.calc.js（純粋計算関数）
  │     └── lib/html5-qrcode.min.js（バーコード読取）
  ├── Service Worker（sw.js）
  │     └── アセットキャッシュ → 完全オフライン動作
  ├── IndexedDB（TanaDB）
  │     ├── products（商品マスタ）
  │     ├── stock_transactions（入出庫履歴）
  │     ├── inventory_counts（棚卸データ）
  │     └── app_settings（設定）
  └── manifest.json（PWAマニフェスト）

[Docker / nginx:alpine]
  └── 静的ファイル配信（ポート 8088）
```

### 3.2 データフロー

1. ユーザーがブラウザから `http://<host>:8088/` にアクセス
2. nginx が静的ファイル（HTML/CSS/JS）を配信
3. Service Worker が全アセットをキャッシュ（以降オフラインで動作可能）
4. アプリケーション起動時に IndexedDB（`TanaDB` v1）を開く
5. 全データの読み書きは IndexedDB に対して行われる
6. 外部サーバーへの通信は一切発生しない

### 3.3 IndexedDB スキーマ

**データベース名:** `TanaDB`
**バージョン:** 1

| オブジェクトストア | keyPath | インデックス |
|-------------------|---------|-------------|
| `products` | `id` | `productCode`（unique）、`janCode`、`name`、`category`、`isActive` |
| `stock_transactions` | `id` | `productId`、`transactionType`、`date` |
| `inventory_counts` | `id` | `countDate`、`status` |
| `app_settings` | `id` | なし |

## 4. 機能要件 — ダッシュボード

**タブ名:** ダッシュボード（`data-tab="dashboard"`）

アプリ起動時のデフォルト表示画面。在庫状況を一覧し、日常的に必要な操作へのクイックアクセスを提供する。

### 4.1 在庫概要

- 登録商品数（有効な商品の総数）を表示する
- 総在庫数（全商品の在庫合計）を表示する

### 4.2 低在庫アラート

- 現在の在庫数が商品の `minStock`（最低在庫数）を下回っている商品の一覧を表示する
- アラート件数をバッジで表示する
- 低在庫の商品がない場合は「在庫不足の商品はありません」を表示する
- 判定ロジック: `calculateCurrentStock(transactions) < product.minStock`

### 4.3 使用期限アラート

- `warning`（期限が近い）または `expired`（期限切れ）のステータスを持つロットの一覧を表示する
- アラート件数をバッジで表示する
- 該当するロットがない場合は「期限が近い商品はありません」を表示する
- 判定ロジック: `getExpiryStatus(expiryDate, product.expiryAlertDays, today)`
  - `expired`: 使用期限日 < 基準日
  - `warning`: 使用期限日 <= 基準日 + アラート日数
  - `ok`: 上記以外

### 4.4 最近の入出庫サマリー

- 直近の入出庫取引を一覧表示する
- 履歴がない場合は「最近の入出庫はありません」を表示する

### 4.5 バックアップリマインダー

- バックアップの状態を表示する
- 設定画面の通知設定と連動する

### 4.6 クイックアクション

以下のボタンを提供し、各機能へ素早くアクセスできるようにする。

| ボタン | 動作 |
|--------|------|
| スキャンして入庫 | バーコードスキャンを起動し、スキャンした商品の入庫フォームを開く |
| スキャンして使用 | バーコードスキャンを起動し、スキャンした商品の使用フォームを開く |
| 棚卸を開始 | 棚卸タブに切り替えて新規棚卸を開始する |

## 5. 機能要件 — 商品管理

**タブ名:** 商品（`data-tab="products"`）

商品マスタの登録・閲覧・編集・削除を行う画面。

### 5.1 商品一覧

- 有効な商品（`isActive=true`）のみを一覧表示する
- 各商品には以下の情報を表示する: 商品コード、商品名、カテゴリ、在庫数、単位、最低在庫数
- 商品をタップすると商品詳細オーバーレイを表示する
- 商品件数を表示する（例: 「10件の商品」）

### 5.2 検索機能

以下のフィールドに対する部分一致（大文字小文字区別なし）で検索を行う。

- 商品名（`name`）
- ふりがな（`nameKana`）
- JANコード（`janCode`）
- 商品コード（`productCode`）

### 5.3 カテゴリフィルター

ドロップダウンで以下のフィルターを選択できる。

| 値 | 表示名 | 動作 |
|----|--------|------|
| `""` (空) | すべて | 全カテゴリを表示 |
| `consumable` | 消耗品 | 消耗品のみ表示 |
| `retail` | 物販 | 物販のみ表示 |

### 5.4 商品登録（新規作成）

- 「商品追加」ボタンで商品フォームオーバーレイを開く
- 商品コードは自動採番される（P-0001 ～ P-9999、既存の最大番号 + 1）
- IDはアプリケーション側で自動生成される（タイムスタンプ + ランダム文字列）
- バリデーション通過後に IndexedDB の `products` ストアに保存する
- 保存成功時にトースト通知を表示する

### 5.5 商品編集

- 商品詳細オーバーレイの「編集」ボタンで商品フォームオーバーレイを開く
- 既存データがフォームに事前入力される
- 商品コードは変更不可（readonly）

### 5.6 商品削除（ソフトデリート）

- 商品詳細オーバーレイの「削除」ボタンで確認ダイアログを表示する
- 確認後、`isActive=false` に設定して保存する（物理削除ではない）
- ソフトデリートされた商品は一覧に表示されないが、データは保持される
- 過去の取引履歴から参照可能な状態を維持する

### 5.7 バーコードスキャン

- 商品一覧画面のFABボタン（「スキャン」）でバーコードスキャンオーバーレイを起動する
- html5-qrcode ライブラリを使用してカメラからバーコードを読み取る
- 対応形式: JAN-8（8桁）、JAN-13（13桁）
- スキャンした JANコードで商品を検索し、該当商品があれば商品詳細を表示する
- 該当商品がない場合は新規登録を提案する

### 5.8 写真撮影・保存

- 商品フォームで端末のカメラまたはギャラリーから画像を取得する
- Canvas APIで画像を圧縮する
  - 最大幅: 400px（アスペクト比維持でリサイズ）
  - JPEG品質: 0.6
  - base64形式で保存
- 写真プレビューを表示する
- 「写真を削除」ボタンで写真をクリアできる

### 5.9 商品コード自動採番ルール

- 形式: `P-NNNN`（Pハイフン4桁ゼロ埋め数字）
- 範囲: P-0001 ～ P-9999
- ロジック: 既存の全商品コードから最大番号を取得し、+1 した値を採番する
- 上限到達時: P-9999 を超えるとエラーを発生させる

## 6. 機能要件 — 入出庫管理

**タブ名:** 入出庫（`data-tab="transactions"`）

商品の入庫・使用・販売を記録し、取引履歴を閲覧する画面。

### 6.1 取引種別

| 種別 | コード | 在庫への影響 | 説明 |
|------|--------|-------------|------|
| 入庫 | `receive` | +（増加） | 商品の入荷・仕入れ |
| 使用 | `use` | -（減少） | 施術等での使用 |
| 販売 | `sell` | -（減少） | 顧客への販売 |
| 調整 | `adjust` | 符号付き（+/-） | 棚卸差異等の在庫調整 |
| 廃棄 | `dispose` | -（減少） | 期限切れ等による廃棄 |

### 6.2 サブタブ構成

| サブタブ | 表示名 | 機能 |
|----------|--------|------|
| `receive` | 入庫 | 入庫フォーム |
| `use` | 使用 | 使用フォーム |
| `sell` | 販売 | 販売フォーム |
| `history` | 履歴 | 取引履歴一覧 |

### 6.3 入庫フォーム

| 項目 | 入力タイプ | 必須 | 説明 |
|------|-----------|------|------|
| 商品 | セレクトボックス | 必須 | 有効商品のドロップダウン |
| 数量 | 数値（min=1, step=1） | 必須 | 入庫数量 |
| 入庫日 | 日付 | 必須 | デフォルト: 当日 |
| 仕入単価 | 数値（min=0） | 任意 | 1個あたりの仕入価格 |
| ロット番号 | テキスト | 任意 | 商品の `trackExpiry=true` 時のみ表示 |
| 使用期限 | 日付 | 任意 | 商品の `trackExpiry=true` 時のみ表示 |
| 備考 | テキストエリア | 任意 | 自由記述（1000文字以内） |

- バーコードスキャンボタンでスキャンした商品を自動選択できる

### 6.4 使用フォーム

| 項目 | 入力タイプ | 必須 | 説明 |
|------|-----------|------|------|
| 商品 | セレクトボックス | 必須 | 有効商品のドロップダウン |
| 数量 | 数値（min=1, step=1） | 必須 | 使用数量 |
| 使用日 | 日付 | 必須 | デフォルト: 当日 |
| 備考 | テキストエリア | 任意 | 自由記述（1000文字以内） |

### 6.5 販売フォーム

| 項目 | 入力タイプ | 必須 | 説明 |
|------|-----------|------|------|
| 商品 | セレクトボックス | 必須 | 有効商品のドロップダウン |
| 数量 | 数値（min=1, step=1） | 必須 | 販売数量 |
| 販売日 | 日付 | 必須 | デフォルト: 当日 |
| 備考 | テキストエリア | 任意 | 自由記述（1000文字以内） |

### 6.6 取引履歴

- 全取引履歴を一覧表示する
- 以下のフィルターを提供する

| フィルター | 入力タイプ | 説明 |
|-----------|-----------|------|
| 開始日 | 日付 | 指定日以降の取引を表示 |
| 終了日 | 日付 | 指定日以前の取引を表示 |
| 商品 | セレクトボックス | 特定商品の取引のみ表示（「すべて」で全商品） |
| 種別 | セレクトボックス | 入庫/使用/販売で絞り込み（「すべて」で全種別） |

- 履歴がない場合は「履歴はありません」を表示する

### 6.7 バーコードスキャン入出庫

- 入出庫タブのFABボタンおよび各フォームのスキャンボタンからバーコードスキャンを起動できる
- スキャンしたJANコードで商品を検索し、該当商品をフォームに自動入力する

### 6.8 ロット番号・使用期限追跡

- 商品の `trackExpiry` が `true` の場合のみ、入庫フォームにロット番号と使用期限の入力欄を表示する
- ロット別在庫は `calculateStockByLot()` で集計する
- 同一ロット番号の取引はグループ化し、ロット単位で在庫数を管理する

## 7. 機能要件 — 棚卸

**タブ名:** 棚卸（`data-tab="inventory"`）

全有効商品の実在庫を確認し、システム在庫との差異を記録・調整する。

### 7.1 新規棚卸の開始

- 「新規棚卸を開始」ボタンで棚卸セッションを作成する
- 全有効商品（`isActive=true`）を棚卸対象としてリストアップする
- 各商品のシステム在庫数を自動計算して初期値とする
- 棚卸セッションは `inventory_counts` ストアに保存される

### 7.2 テンキーによる実数入力

- 商品をタップするとテンキーオーバーレイを表示する
- テンキーオーバーレイには以下を表示する
  - 商品名
  - 商品写真（登録されている場合）
  - システム在庫数
  - 入力中の数値（ディスプレイ）
- テンキーの操作

| ボタン | 動作 |
|--------|------|
| 0〜9 | 数値入力（ディスプレイに追記） |
| C | 入力クリア（ディスプレイを0に戻す） |
| ← | バックスペース（末尾1桁削除） |
| 確定 | 入力値を実在庫数として保存 |
| キャンセル | 入力を破棄してオーバーレイを閉じる |

### 7.3 進捗バー

- カウント済み商品数 / 全対象商品数 をパーセンテージで表示する
- プログレスバーとパーセンテージテキストで視覚的に進捗を示す

### 7.4 棚卸完了と差異レポート

- 「棚卸を完了」ボタンで棚卸を完了する
- 完了時に差異レポートを自動生成する
  - レポート内容: 全商品数、カウント済み数、差異ありの商品数、プラス差異合計、マイナス差異合計、各商品の詳細
- 差異がある商品に対して、`adjust`（調整）タイプの取引を自動生成する
  - 調整数量: 実在庫数 - システム在庫数
  - 備考: 「棚卸調整」
- 差異レポートの計算ロジック: `generateVarianceReport()` 関数

> **備考（冪等性）:** 棚卸完了後、UIは `activeCountId` をクリアすることで再完了操作を防止する。コードレベルでの冪等性ガードは設けておらず、UI状態管理に依拠する設計としている。

### 7.5 棚卸履歴

- 過去の棚卸セッション一覧を表示する
- 棚卸履歴がない場合は「棚卸履歴はありません」を表示する

## 8. 機能要件 — レポート

**タブ名:** レポート（`data-tab="reports"`）

在庫データを多角的に分析・確認するためのレポート機能。

### 8.1 サブタブ構成

| サブタブ | 表示名 | 機能 |
|----------|--------|------|
| `report-stock` | 在庫一覧 | 全有効商品の在庫サマリー |
| `report-history` | 入出庫履歴 | 取引履歴のフィルター付きレポート |
| `report-expiry` | 使用期限 | ロット別期限ステータス一覧 |
| `report-variance` | 棚卸差異 | 棚卸セッション別の差異レポート |

### 8.2 在庫一覧レポート

全有効商品の在庫状況を一覧表示する。

- 表示項目: 商品ID、商品名、商品コード、カテゴリ、現在在庫数、在庫金額、単位
- フィルター: カテゴリ（すべて/消耗品/物販）
- 並び替え: 商品名、在庫数（少ない順）、在庫数（多い順）、在庫金額
- 在庫金額の計算: 入庫取引の加重平均原価 × 現在在庫数（`calculateStockValue()`）
- データがない場合は「データがありません」を表示する

### 8.3 入出庫履歴レポート

取引履歴を日付範囲・商品・種別で絞り込んで表示する。

- フィルター

| フィルター | 入力タイプ | 説明 |
|-----------|-----------|------|
| 開始日 | 日付 | 指定日以降の取引を表示 |
| 終了日 | 日付 | 指定日以前の取引を表示 |

| `ok` | 期限日 > 基準日 + アラート日数 | 問題なし |

## 12. データバリデーションルール

全バリデーション関数は `tana.calc.js` に実装されている純粋関数であり、DOM操作やDB操作を含まない。

### 12.1 商品バリデーション（`validateProduct()`）

| フィールド | ルール | エラーメッセージ |
|-----------|--------|----------------|
| `name` | 必須、文字列型、`trim()` 後の長さが1以上 | 「商品名は必須です」 |
| `name` | 100文字以内 | 「商品名は100文字以内で入力してください」 |
| `category` | `"consumable"` または `"retail"` のいずれか | 「カテゴリは「consumable」または「retail」を指定してください」 |
| `janCode` | 空値は許容。空でない場合は `validateJanCode()` で検証 | 「JANコードが不正です」 |
| `nameKana` | 空値は許容。空でない場合はひらがな・長音符・スペースのみ | 「フリガナはひらがなで入力してください」 |
| `defaultPrice` | 空値は許容。空でない場合は0以上の数値 | 「販売価格は0以上の数値を入力してください」 |
| `costPrice` | 空値は許容。空でない場合は0以上の数値 | 「原価は0以上の数値を入力してください」 |
| `minStock` | 空値は許容。空でない場合は0以上の数値 | 「最小在庫数は0以上の数値を入力してください」 |
| `expiryAlertDays` | 空値は許容。空でない場合は0以上の数値 | 「期限アラート日数は0以上の数値を入力してください」 |

**ふりがなの正規表現:** `/^[\u3040-\u309F\u30FC\s]+$/`
- `\u3040-\u309F`: ひらがな（ぁ〜ゟ）
- `\u30FC`: カタカナ長音符（ー）
- `\s`: 空白文字（半角・全角スペース等）

### 12.2 取引バリデーション（`validateTransaction()`）

| フィールド | ルール | エラーメッセージ |
|-----------|--------|----------------|
| `productId` | 必須（空でないこと） | 「商品IDは必須です」 |
| `transactionType` | `receive`/`use`/`sell`/`adjust`/`dispose` のいずれか | 「取引種別が不正です（receive, use, sell, adjust, dispose）」 |
| `quantity` | 必須、`null`/`undefined` でないこと | 「数量は必須です」 |
| `quantity` | 0以外の数値（`NaN` は不可） | 「数量は0以外の数値を入力してください」 |
| `date` | 必須（空でないこと） | 「日付は必須です」 |
| `date` | `YYYY-MM-DD` 形式（正規表現: `/^\d{4}-\d{2}-\d{2}$/`） | 「日付はYYYY-MM-DD形式で入力してください」 |
| `notes` | 空値は許容。空でない場合は1000文字以内 | 「備考は1000文字以内で入力してください」 |

### 12.3 JANコードバリデーション（`validateJanCode()`）

JANコードの検証は以下の手順で行う。

1. **空値チェック**: `null`、`undefined`、空文字列の場合は有効（任意項目のため）
2. **数字チェック**: 数字のみで構成されていること（正規表現: `/^\d+$/`）
3. **桁数チェック**: 8桁（JAN-8）または 13桁（JAN-13）であること
4. **チェックデジット検証**:

**JAN-13 のチェックデジット計算:**
```
sum = Σ(i=0..11) digit[i] × weight[i]
weight = [1, 3, 1, 3, 1, 3, 1, 3, 1, 3, 1, 3]
checkDigit = (10 - (sum % 10)) % 10
有効条件: checkDigit === digit[12]
```

**JAN-8 のチェックデジット計算:**
```
sum = Σ(i=0..6) digit[i] × weight[i]
weight = [3, 1, 3, 1, 3, 1, 3]
checkDigit = (10 - (sum % 10)) % 10
有効条件: checkDigit === digit[7]
```

### 12.4 インポートデータバリデーション（`validateImportData()`）

| チェック項目 | ルール | エラーメッセージ |
|-------------|--------|----------------|
| データ全体 | `null`/`undefined` でないこと | 「インポートデータが空です」 |
| `appName` | `"tana"` であること | 「appNameが"tana"ではありません」 |
| `products` | 配列であること | 「productsが配列ではありません」 |
| `products[n].id` | 各要素に `id` が存在すること | 「products[N]にidがありません」 |
| `stock_transactions` | 配列であること | 「stock_transactionsが配列ではありません」 |
| `inventory_counts` | 配列であること | 「inventory_countsが配列ではありません」 |
| `settings` | 存在する場合はオブジェクトまたは配列であること | 「settingsはオブジェクトまたは配列である必要があります」 |

> **既知の制限（参照整合性）:** インポート時に参照整合性の検証は行わない。例えば `stock_transactions[n].productId` が `products` 内の既存商品IDを参照しているかどうかはチェックしない。これは設計上の既知の制限である。

| 制約 | 理由 |
|------|------|
| 外部ライブラリの追加禁止 | html5-qrcode のみ例外として許可。依存関係を最小化しメンテナンス性を維持するため |
| サーバーサイド処理なし | アプリケーションは完全にクライアントサイドで動作する。外部APIやデータベースサーバーは使用しない |
| `tana.calc.js` は純粋関数のみ | DOM操作・IndexedDB操作を禁止し、テスタビリティと再利用性を確保する |
| HTML特殊文字のエスケープ必須 | ユーザー入力値をDOMに挿入する際は、必ず `escapeHtml()` を使用する |
| 商品コード上限 | P-9999 が上限（最大9,999商品のコード採番） |

### 14.2 データに関する前提

| 前提 | 説明 |
|------|------|
| データはブラウザ内に保存される | ブラウザのデータ消去やキャッシュクリアでデータが失われる可能性がある |
| 端末間のデータ同期なし | 複数端末でのリアルタイム同期は提供しない（エクスポート/インポートによる手動移行のみ） |
| マルチユーザー非対応 | 単一端末での利用を前提とする。ユーザー認証機能は提供しない |

### 14.3 運用上の前提

| 前提 | 説明 |
|------|------|
| 定期バックアップの推奨 | ユーザーは定期的にデータをエクスポートしてバックアップを取得することを推奨する |
| バーコードスキャンにはカメラが必要 | バーコードスキャン機能を利用するには、端末にカメラが搭載されている必要がある |
| HTTPS環境が望ましい | Service Worker やカメラアクセスはセキュアコンテキスト（HTTPS または localhost）で動作する |

## 15. 用語集

| 用語 | 説明 |
|------|------|
| PWA（Progressive Web App） | Webアプリケーションにネイティブアプリのような機能（オフライン動作、ホーム画面追加等）を付加する技術 |
| SPA（Single Page Application） | ページ遷移なしに、JavaScript でコンテンツを動的に切り替えるWebアプリケーションの設計パターン |
| IndexedDB | ブラウザに組み込まれた大容量の構造化データストレージ |
| Service Worker | バックグラウンドで動作するスクリプト。ネットワークリクエストの制御やキャッシュ管理を行う |
| JAN コード | Japanese Article Number。日本で標準的に使用される商品バーコード規格。JAN-8（8桁）と JAN-13（13桁）がある |
| チェックデジット | バーコードの読み取りエラーを検出するための検証用数字。コードの末尾1桁がこれにあたる |
| ソフトデリート | データベースからレコードを物理的に削除せず、フラグ（`isActive=false`）で無効化する削除方式 |
| ロット番号 | 同一条件で製造された製品群を識別するための番号 |
| 加重平均原価 | 各仕入時の単価と数量を加味して算出する平均仕入原価。在庫金額の計算に使用する |
| トースト通知 | 画面の端に一時的に表示され、自動的に消える非モーダルな通知メッセージ |
| FAB（Floating Action Button） | 画面上に浮いた状態で表示される円形のアクションボタン |
| base64 | バイナリデータをASCII文字列で表現するエンコーディング方式。写真データの保存に使用する |
